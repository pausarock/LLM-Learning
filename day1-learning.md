## 2. 核心概念讲解

### 2.1 为什么需要注意力？

**直觉**：模型在表示某个位置时，需要「看向」句子里其他位置。长距离依赖若只靠 RNN 逐步传递，容易衰减；注意力允许**任意位置对之间直接建立联系**（再通过堆叠层数组合高阶关系）。

### 2.2 Query、Key、Value

| 概念 | 含义（建模视角） |
|------|------------------|
| **Query** | 当前位置「在问什么」 |
| **Key** | 每个位置「可被匹配的标签」 |
| **Value** | 匹配成功后「取回的内容向量」 |

**检索类比**：Query = 搜索词；Key = 标题/标签；Value = 正文。注意力权重由 \(Q\) 与 \(K\) 的匹配程度得到，再对 \(V\) 加权求和。

### 2.3 Scaled Dot-Product Attention

对某一组 Query、Key、Value（行向量为词向量，实际实现多为批量矩阵）：

\[
\mathrm{Attention}(Q,K,V)=\mathrm{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right)V
\]

**步骤**：

1. **打分**：\(QK^\top\) 得到每个 query 对每个 key 的相似度（点积）。
2. **缩放**：除以 \(\sqrt{d_k}\)。当维度 \(d_k\) 较大时，点积方差增大，softmax 易进入**极端饱和区**，梯度变小；缩放使分布更温和。
3. **归一**：softmax 得到权重，行和为 1。
4. **聚合**：权重乘 \(V\) 并求和，得到输出表示。

### 2.4 自注意力（Self-Attention）

**同一序列**先嵌入为 \(X\)，再经三个线性层得到 \(Q=XW^Q,\;K=XW^K,\;V=XW^V\)。  
因此模型既能学「谁和谁相关」，又能学「取回什么信息」。

### 2.5 矩阵形状（必读）

设 batch 大小 \(B\)，序列长度 \(L\)，模型维度 \(d_{\mathrm{model}}\)，单头 key 维度 \(d_k\)：

- \(Q,K,V\) 常见形状：\((B,\,L,\,d_k)\) 或多头时先 \((B,\,L,\,d_{\mathrm{model}})\) 再拆头  
- 注意力分数：\(QK^\top\) → \((B,\,L,\,L)\)（每个 query 位置对每个 key 位置一个分数）  
- 输出：与 \(V\) 同型 \((B,\,L,\,d_k)\)

### 必做 2：手算「喜欢」的输出（与教材一致）

句子「我 / 喜欢 / 猫」，词向量（2 维）：

- 我：\([1,0]\)  
- 喜欢：\([0,1]\)  
- 猫：\([1,1]\)  

令 **单头**且 **\(Q=K=V\)** = 上述三行堆成的矩阵（每行一个词）。取 **Query 为「喜欢」那一行**，对三个位置做 scaled dot-product attention，\(d_k=2\)。

要求写出：**未缩放 logits → 除以 \(\sqrt{2}\) → softmax 权重 → 加权和向量**（可与原 day1 核对，但必须自己先算一遍）。

<details>
<summary>参考答案（要点）</summary>

- 点积：与「我」0，与「喜欢」1，与「猫」1  
- 除以 \(\sqrt{2}\)：约 \([0,\,0.707,\,0.707]\)  
- softmax：约 \([0.23,\,0.39,\,0.39]\)  
- 输出：\(0.23[1,0]+0.39[0,1]+0.39[1,1]\approx[0.62,\,0.78]\)

</details>

### 必做 4：简答题

1. 不除以 \(\sqrt{d_k}\) 时，**\(d_k\) 很大** 会对 softmax 和训练带来什么现象？（各一句话）  
2. **自注意力** 与 **交叉注意力**（先只查定义）最关键的一点差别是什么？

<details>
<summary>参考答案</summary>

1. 点积数值方差随维度增大，softmax 更易极端（接近 one-hot），梯度变小、不稳定。  
2. 自注意力：Q、K、V 来自**同一序列**；交叉注意力：Q 来自一方序列，K/V 来自**另一方**（如解码器看编码器输出）。

</details>

### 选做 5：10 行以内 NumPy 验证

用 `numpy` 复现必做 2：构造 \(X\)（3×2），令 \(Q=K=V=X\)，实现 `softmax(x, axis=-1)`，计算 `softmax((Q @ K.T) / np.sqrt(2)) @ V`，检查第二行输出是否接近手算。

<details>
<summary>参考代码</summary>

```python
import numpy as np

def softmax(x, axis=-1):
    x = x - x.max(axis=axis, keepdims=True)
    e = np.exp(x)
    return e / e.sum(axis=axis, keepdims=True)

X = np.array([[1., 0.], [0., 1.], [1., 1.]])
dk = X.shape[1]
scores = (X @ X.T) / np.sqrt(dk)
weights = softmax(scores, axis=1)
out = weights @ X
print(out[1])  # 「喜欢」对应第 2 行（0-based 索引为 1）
```

</details>

### 进阶 6：形状推断

\(B=2,\,L=5,\,d_k=64\)。\(Q,K,V\) 均为 \((B,L,d_k)\)。

- \(QK^\top\) 的形状？  
- softmax 在哪个维度上做？  
- 与 \(V\) 相乘后输出形状？

<details>
<summary>参考答案</summary>

- \(QK^\top\)：\((B,\,L,\,L)\)（常实现为对最后两维做 batched matmul）  
- softmax 在 **最后一个维度**（对每个 query 位置，在 key 位置上归一化）  
- 输出：\((B,\,L,\,d_k)\)

</details>
