# 大语言模型专用术语表（自学整理）

> 按主题分类，便于背诵与检索。每条为 **一句话** 级定义；面试可再补「例子 / 取舍 / 坑」。

---

## 1. 基础架构与表示

| 英文/缩写 | 中文参考 | 一句话解释 |
|-----------|----------|------------|
| **Token** | 词元 | 模型处理的最小单位；由分词器切分，不等同于「字」。 |
| **Tokenizer** | 分词器 | 字符串 ↔ token id；BPE/SentencePiece 等子词算法压缩词表、处理 OOV。 |
| **BPE / SentencePiece** | 字节对编码等 | 从语料统计合并高频子串，平衡词表大小与序列长度。 |
| **Vocabulary (vocab)** | 词表 | token 到 id 的映射表；影响多语种与特殊符号。 |
| **Embedding** | 嵌入 | 离散 token 映射为稠密向量；可与位置信息相加或复合。 |
| **d_model** | 模型宽度 | 每层主隐藏维度；与头数、FFN 中间层一起决定容量。 |
| **Transformer** | 变换器 | 以自注意力为核心堆叠的深度网络；现代 LLM 主干。 |
| **Self-Attention** | 自注意力 | Q、K、V 来自同一序列；建模 token 之间任意依赖。 |
| **Cross-Attention** | 交叉注意力 | Q 来自一方序列，K/V 来自另一方（如解码器看编码器）。 |
| **Multi-Head Attention** | 多头注意力 | 多组并行注意力子空间再拼接；丰富关系类型。 |
| **Scaled Dot-Product Attention** | 缩放点积注意力 | softmax(QK^T/√d_k)V；缩放避免高维点积导致 softmax 饱和。 |
| **Causal / Autoregressive** | 因果 / 自回归 | 只能看当前及过去 token；GPT 类生成模型。 |
| **Bidirectional** | 双向 | 可同时看整句上下文；BERT 编码器 + MLM。 |
| **Encoder** | 编码器 | 把输入序列编码为上下文向量；理解任务常用。 |
| **Decoder** | 解码器 | 自左向右生成；常含掩码自注意力与（可选）交叉注意力。 |
| **Encoder-Decoder** | 编解码结构 | Seq2Seq（如翻译）；两侧各一堆叠层。 |
| **FFN / MLP** | 前馈网络 | 逐位置两层（通常扩 4 倍再压回）非线性变换；与注意力交替。 |
| **Residual Connection** | 残差连接 | x + Sublayer(x)；缓解梯度消失、利于堆深。 |
| **Layer Normalization** | 层归一化 | 在特征维上归一化单个样本；Transformer 常用 Pre-LN 或 Post-LN。 |
| **Positional Encoding** | 位置编码 | 注入顺序信息；正弦、可学习、RoPE 等。 |
| **RoPE** | 旋转位置编码 | 对 Q/K 施加位置相关旋转；常见于 LLaMA 系，利于相对位置与外推。 |
| **ALiBi** | 注意力线性偏置 | 用距离偏置替代显式位置表；外推讨论中常见。 |
| **GQA / MQA** | 分组/多查询注意力 | 多组 Q 共享 K/V 或单组 KV；减少 KV Cache、省显存。 |

## 2. 预训练、微调与对齐

| 英文/缩写 | 中文参考 | 一句话解释 |
|-----------|----------|------------|
| **Pre-training** | 预训练 | 大规模无标注数据上学习通用表示；如 CLM、MLM。 |
| **CLM** | 因果语言建模 | 预测下一 token；GPT 类目标。 |
| **MLM** | 掩码语言建模 | 预测被遮盖 token；BERT 类。 |
| **SFT** | 监督微调 | 用指令–回答等配对数据微调，学任务格式与行为。 |
| **IFT / Instruction Tuning** | 指令微调 | 与 SFT 常混用；强调「遵循指令」数据。 |
| **PEFT** | 参数高效微调 | 只训少量参数（适配器、LoRA 等），冻结大部分权重。 |
| **LoRA** | 低秩适配 | W' = W + BA，低秩更新；省显存与存储。 |
| **QLoRA** | 量化 LoRA | 4bit 等量化基底 + LoRA；单卡可训更大模型。 |
| **Adapter** | 适配器 | 层间小模块；PEFT 一种。 |
| **Prefix-Tuning / P-Tuning** | 前缀/连续提示微调 | 训可学习前缀或连续向量，少改原权重。 |
| **Full Fine-tuning** | 全量微调 | 更新全部权重；效果好但贵、易灾难性遗忘。 |
| **Catastrophic Forgetting** | 灾难性遗忘 | 微调后通用能力下降；可用回放、正则、LoRA 等缓解。 |
| **RLHF** | 人类反馈强化学习 | 奖励模型 + PPO 等，对齐人类偏好。 |
| **Reward Model (RM)** | 奖励模型 | 学人类排序/评分，为 RL 提供标量奖励。 |
| **PPO** | 近端策略优化 | RLHF 中常用策略梯度算法。 |
| **DPO** | 直接偏好优化 | 用偏好对直接优化，避免显式 RM + PPO 的复杂管线。 |
| **RLAIF** | AI 反馈强化学习 | 用 AI 替代部分人类标注的偏好信号。 |
| **Alignment** | 对齐 | 使模型更安全、有用、诚实；SFT+RLHF/DPO 等。 |
| **Hallucination** | 幻觉 | 生成看似合理但事实错误或无依据的内容。 |
| **Grounding** | 接地 / 有据 | 输出可追溯到给定上下文或检索片段。 |

## 3. 推理、效率与部署

| 英文/缩写 | 中文参考 | 一句话解释 |
|-----------|----------|------------|
| **Inference** | 推理 | 前向计算生成输出；与训练相对。 |
| **KV Cache** | KV 缓存 | 解码时缓存历史 K/V，避免重复计算已生成部分。 |
| **Prefill** | 预填充阶段 | 对整段 prompt 并行计算，常占首包延迟大头。 |
| **Decode** | 解码阶段 | 逐 token 自回归生成；常受内存带宽限制。 |
| **TTFT** | 首 token 时间 | 从请求到首个输出 token 的延迟。 |
| **Throughput (TPS)** | 吞吐 | 每秒生成 token 数；与批大小、量化等相关。 |
| **Batching** | 批处理 | 合并请求提高 GPU 利用率；动态批可减尾延迟。 |
| **Continuous Batching** | 连续批处理 | vLLM 等动态合批，提高吞吐。 |
| **PagedAttention** | 分页注意力 | vLLM 将 KV 分页管理，减碎片、提并发。 |
| **Quantization** | 量化 | 降低权重/激活精度（INT8/INT4 等）以省显存、加速。 |
| **PTQ** | 训练后量化 | 训练完成后量化权重；需校准数据。 |
| **QAT** | 量化感知训练 | 训练中模拟量化，精度通常优于纯 PTQ。 |
| **GPTQ / AWQ / GGUF** | 常见量化方案 | 不同权重量化与打包格式；部署生态各异。 |
| **Distillation** | 蒸馏 | 用大模型（教师）信号训练小模型（学生）。 |
| **Speculative Decoding** | 投机解码 | 小模型草稿 + 大模型并行验证，提高有效速度。 |
| **FlashAttention** | 闪存注意力 | 分块减少 HBM 读写，加速长序列注意力。 |
| **MoE** | 混合专家 | 多专家前馈，路由激活部分专家；总参大、激活算力可控。 |
| **OpenAI-compatible API** | 兼容接口 | `/v1/chat/completions` 等约定；便于切换后端。 |
| **vLLM / TGI / TensorRT-LLM** | 推理服务栈 | 高吞吐服务框架；应用岗需知选型与限制。 |

## 4. 提示、上下文与应用模式

| 英文/缩写 | 中文参考 | 一句话解释 |
|-----------|----------|------------|
| **Prompt** | 提示 | 输入给模型的文本/多段消息；含 system/user/assistant。 |
| **System Prompt** | 系统提示 | 设定角色、边界、工具策略；需防注入篡改。 |
| **Few-shot / Zero-shot** | 少样本 / 零样本 | 提示里是否给示例；影响 ICL 效果。 |
| **ICL** | 上下文学习 | 不更新权重，靠提示内示例完成任务。 |
| **CoT** | 思维链 | 提示模型分步推理，提升复杂推理题表现。 |
| **Temperature / Top-p / Top-k** | 采样超参 | 控制随机性；温度高更发散，低更确定。 |
| **Max Tokens** | 最大生成长度 | 限制输出长度与费用。 |
| **Stop Sequences** | 停止序列 | 遇到特定串即停。 |
| **Streaming** | 流式输出 | SSE 等边生成边返回；改善首字体验。 |
| **Structured Output** | 结构化输出 | JSON Schema、工具参数等可解析格式。 |
| **Function / Tool Calling** | 函数/工具调用 | 模型输出结构化调用，由宿主执行并返回 observation。 |
| **ReAct** | 推理+行动 | Thought–Action–Observation 循环范式。 |
| **Agent** | 智能体 | 多步使用工具、记忆、规划完成目标的系统。 |
| **Planner–Executor** | 规划–执行 | 先分解任务再逐步执行的一类 Agent 结构。 |
| **Human-in-the-loop** | 人机回环 | 关键步骤需人确认；降风险。 |

## 5. RAG 与知识增强

| 英文/缩写 | 中文参考 | 一句话解释 |
|-----------|----------|------------|
| **RAG** | 检索增强生成 | 检索相关文档片段再生成，补知识、可引用。 |
| **Chunking** | 分块 | 文档切为检索单元；大小与重叠影响召回与上下文。 |
| **Embedding Model** | 嵌入模型 | 将文本编码为向量；与生成模型可分离选型。 |
| **Vector Database** | 向量数据库 | 存储与检索高维向量；常 ANN 索引。 |
| **ANN** | 近似最近邻 | 牺牲少量精度换速度；HNSW、IVF 等。 |
| **BM25** | 稀疏检索 | 经典词项统计检索；可与向量融合。 |
| **Hybrid Search** | 混合检索 | 稀疏 + 稠密融合，提高召回与可解释性。 |
| **Reranker** | 重排序 | 二阶段用交叉编码器等精排 Top-K。 |
| **MRR / nDCG / Hit@k** | 检索指标 | 排序质量评估；线上可配业务指标。 |
| **Query Rewriting** | 查询改写 | 把用户问句改写成更利于检索的表达。 |
| **HyDE** | 假设文档嵌入 | 先生成假设答案再嵌入检索，缓解词汇鸿沟。 |
| **Faithfulness / Groundedness** | 忠实度 | 答案是否被检索内容支持；RAG 评测核心。 |
| **Citation / Provenance** | 引用 / 出处 | 回答标注来源片段 id，便于审计。 |

## 6. 安全、合规与运维

| 英文/缩写 | 中文参考 | 一句话解释 |
|-----------|----------|------------|
| **Prompt Injection** | 提示注入 | 用户内容操纵模型违背系统指令或泄露秘密。 |
| **Jailbreak** | 越狱 | 绕过安全对齐的提示技巧。 |
| **PII** | 个人可识别信息 | 需脱敏与访问控制。 |
| **RBAC** | 基于角色的访问控制 | 工具与数据按角色授权。 |
| **Least Privilege** | 最小权限 | 工具默认能力最小化。 |
| **Output Filtering** | 输出过滤 | 敏感内容、格式、长度后置校验。 |
| **Red Teaming** | 红队测试 | 主动找模型与系统漏洞。 |
| **OWASP LLM Top 10** | LLM 十大风险 | 应用安全清单类参考。 |
| **Observability** | 可观测性 | 日志、指标、追踪；含 token、延迟、检索命中。 |
| **Idempotency** | 幂等 | 重试不产生重复副作用；API 设计要点。 |

## 7. 评测与数据

| 英文/缩写 | 中文参考 | 一句话解释 |
|-----------|----------|------------|
| **Benchmark** | 基准 | 标准任务集比较模型能力。 |
| **Perplexity** | 困惑度 | 语言模型概率质量指标；越低通常越好（同域比较）。 |
| **BLEU / ROUGE** | 自动评价 | n-gram 重叠；摘要/翻译传统指标。 |
| **LLM-as-a-Judge** | 模型裁判 | 用大模型打分；有 position bias 等局限。 |
| **Ragas** | RAG 评测框架 | faithfulness、answer relevance 等自动化指标。 |
| **Golden Set** | 黄金集 | 人工标注问答对；用于回归与 A/B。 |
| **Online A/B** | 线上对比 | 真实流量比较策略与模型版本。 |

## 使用建议

1. **每周过一类**：8～12 周可轮两遍；第二遍只背仍卡壳的词条。  
2. **卡片背面写**：一个例子 + 一个「反面坑」。  
3. **与 `learning-guide.md` 周计划对齐**：学到哪周就优先背该周出现的类（如 W2–W3 主攻 §5，W4 主攻 §4）。
